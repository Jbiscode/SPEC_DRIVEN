## 한 줄 정의 (친구들에게 전달용)

| 용어 | 한 줄 정의 |
|------|------------|
| **Rules** | AI가 항상 지켜야 하는 공통 원칙 |
| **Skills** | 반복 업무를 잘하게 만드는 재사용 가능한 작업 모듈 |

---

## 스터디에서 같이 보면 좋은 질문 (4개)

1. 좋은 프롬프트와 rules의 차이는 무엇인가?
2. 어떤 작업이 skill로 분리될 만큼 반복적인가?
3. 팀에서 AI 결과물 편차를 줄이려면 어떤 rules가 필요한가?
4. 개인 노하우를 팀 자산으로 바꾸려면 무엇을 문서화해야 하나?

## 면접 답변 예시 (그대로 활용 가능)

> “저는 AI를 단발성 질문 도구로만 쓰지 않고, 반복 업무는 규칙과 템플릿으로 구조화해서 재사용하려고 했습니다. 예를 들어 API 출력 포맷 규칙을 정하고, 자주 쓰는 작업은 skill처럼 표준화해서 품질 편차를 줄이는 방식으로 활용했습니다.”

**장점**: 단순 사용자처럼 안 보임 · 팀 생산성 관점 · 실무형 사고 표현

---

## 면접에서 같이 묶으면 좋은 키워드 (4개)

| 키워드 | 한 줄 설명 |
|--------|------------|
| **Spec Driven Development** | 구현 전에 명세를 명확히 만드는 방식 |
| **Context Engineering** | AI에게 어떤 문맥을 넣을지 설계하는 것 |
| **Rules** | AI가 항상 따를 원칙 |
| **Skills** | 반복 작업을 재사용 가능하게 만든 단위 |

**한 흐름으로 말할 때:**  
Spec으로 요구를 정리하고, Context를 넣고, Rules로 행동을 통제하고, Skills로 반복 업무를 재사용한다.
