# Part 4: Spec & Context

## Spec-Driven Development
- **요구사항을 Spec(명세)으로 먼저 정리**하고, 그 Spec을 기준으로 개발하는 방식
- AI와 협업할 때 **Spec이 source of truth(진실의 원천)**가 됨

## Context Engineering
- **AI에게 어떤 문맥을 넣을지 설계하는 것**
- "무엇을 알려줄지"를 의도적으로 설계해 품질과 일관성을 높임  

**프롬프트 엔지니어링에서 Context 설계로**  
AI 초기에는 **프롬프트 엔지니어링**(질문을 어떻게 잘 만들지, 어떤 표현을 쓸지)이 매우 중요했습니다. 요즘은 모델이 이해력을 많이 갖춰서, "질문 문장을 예쁘게 다듬는 것"보다 **AI가 그 상황을 제대로 파악할 수 있도록 필요한 컨텍스트를 골라 넣어 주고, 그걸 어떻게 해석하면 좋을지 알려주는 것**이 더 효과적입니다. 즉, **어떤 문맥을 주고, 그 문맥을 어떻게 쓰게 할지**를 설계하는 쪽이 중요해졌습니다.

## Agentic Coding
- AI가 **여러 단계를 스스로 진행**하는 코딩 방식 (할 일 정리 → 코드 작성 → 수정 등)
- 단순 Q&A가 아니라 **작업 흐름 안에서 AI가 에이전트처럼 동작**

## Codebase RAG
- 코드베이스를 검색·참조해서 **문맥에 맞는 답변**을 만들게 하는 방식
- "우리 프로젝트 코드 기준으로" 답하게 만드는 기술
- Codebase RAG는 커서에서 프로젝트를 인덱싱할때 사용하고있다.

**실제로 일할 때는 Rules와 Skills가 품질 편차를 줄여 준다.**
